# buffer pool
 缓冲池是MySQL中最重要的组件之一，它位于内存中，用于缓存磁盘上的数据页。  
 虽然因为原地更新等策略，与LSM Tree的思想冲突，但是也有所保留，比如：change buffer。  
 写入数据时，先写入buffer pool，然后异步刷盘。   
 同时会将数据页的变更记录到redo log中，用于恢复数据。
 即WAL（Write-Ahead Logging）机制。  
 # 锁
* 行锁：锁定单行数据，粒度最小，并发度最高。  
* 表锁：锁定整张表，粒度最大，并发度最低。  
* 间隙锁：锁定索引之间的间隙，防止幻读。  
* 意向锁：表级锁，用于解决死锁问题。  
* 自增锁：用于解决自增主键冲突问题。  
* 读锁：共享锁，用于解决读写冲突问题。  
* 写锁：排他锁，用于解决写写冲突问题。
# 事务隔离级别
* 读未提交（Read Uncommitted）：一个事务可以读取另一个事务未提交的数据。
* 读已提交（Read Committed）：一个事务只能读取另一个事务已提交的数据。
* 可重复读（Repeatable Read）：一个事务在执行过程中多次读取同一数据，结果是一致的。
* 串行化（Serializable）：事务串行执行，避免了幻读问题。  
RC、RR、SERIALIZABLE 都是基于 MVCC 实现的。
