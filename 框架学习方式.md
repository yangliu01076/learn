深度掌握一个框架，绝不仅仅是学会它的 API 调用或者能照着文档写出
一个 Demo。真正的“深度掌握”，意味着你从原理层面理解它，具备源码
级的调试能力，能够对其扩展，甚至能够自己造一个简化的轮子。  
1. 建立宏观认知
* 搞清楚“它解决了什么问题”：
在它出现之前，大家是怎么开发的？有什么痛点？  
例如： React 出现之前，jQuery 时代直接操作 
DOM 导致代码难以维护，React 用 Virtual DOM 和组件化思想解决了这个问题。
* 理解核心设计理念：
框架的“哲学”是什么？是约定优于配置，还是灵活至上？  
例如： Spring 提倡“非侵入式”，Go 语言的标准库提倡“少即是多”。
* 构建知识脑图：
画出架构图，知道核心模块有哪些，它们之间是如何交互
的（数据流向是怎样的）。  
例如： 请求是如何进入框架的？经过了哪些拦截器？最
后是如何渲染并返回的？
